## What this migration does

Repairs the etc harness's prompt surfaces for Claude Opus 4.7's literal
instruction-following. Under 4.6, prompts could gesture ("follow best
practices") and the model inferred. Under 4.7, the model executes
literally — gestures become no-ops.

Scope: every agent, skill, hook prompt, and directive standards doc in
the harness. **192 edits across 68 files.**

Along the way, surfaced and fixed 7 latent defects that pre-dated 4.7
but were tolerated under 4.6 generalization.

## New invariants introduced

### AP-013: Reference without read-enforcement

An agent prompt that references a standards doc without a mechanism
that forces the Read (Before Starting section, `inject-standards.sh`,
or `requires_reading` in task YAML) is unsound under 4.7 — the
reference becomes a false promise. Every agent in the harness now
passes AP-013 structurally.

See `opus-4.7-migration-spec.md` section 3 AP-013 for the full
check procedure.

### Subagent Dispatch Taxonomy

Every skill now carries one of two explicit sections:

- **"Subagent Dispatch (Non-Negotiable)"** — for orchestrators that
  MUST dispatch via the Agent tool. Includes MUST/MUST NOT rules.
- **"Subagent Dispatch (Non-Applicable)"** — for engine, pipeline, or
  CLI-wrapper skills that MUST NOT dispatch. Includes the rationale
  and an explicit list of allowed in-context actions.

The taxonomy prevents the 4.7-literalism failure where a skill
silently tries to become an orchestrator, or an orchestrator silently
executes work in its own context.

### Verbosity Directive Required

Every agent and skill now has an explicit "Response Format
(Verbosity)" section with word caps, formatting rules, and
preamble/emoji bans. Under 4.7's complexity-calibrated verbosity, this
prevents output-length drift.

## How to soak-test

1. Recompile: `python3 compile-sdlc.py spec/etc_sdlc.yaml` (already
   done; `dist/` is current with branch HEAD)
2. Install locally: `./install.sh`
3. Restart Claude Code session to pick up installed changes
4. Use the harness normally for a few hours
5. Watch for: agents skipping required reads, verbosity drift,
   dispatch failures, standards-reference errors, unexpected refusals
   (AP-012 defensive-framing regressions)

If soak passes: merge `4.7-migration` → `main`, tag `v1.8`, push.
If soak fails: `git checkout main && ./install.sh` restores v1.7.

The branch is on-laptop only. `main` is untouched at `v1.7`.
Rollback is trivial.
