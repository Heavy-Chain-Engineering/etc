# ADR F-2026-06-10-brownfield-architecture-baseline-005: Workspace Topology — Per-Repo Baselines + Canonical Seam Map + Read-Only Mirrors

**Date:** 2026-06-11
**Status:** Accepted
**Context:** The covr system spans two repos with three informal cross-repo contracts (stub-per-route URL ownership, window-global session injection, a shared DB whose schema lives in the other repo) — and the "worked locally, broke in dev" failures were exactly seam blindness. A workspace directory containing N repos is not itself a git repo, so the cross-repo artifact needs a home; and a repo cloned alone must not lose its seam context (that solo-clone blindness IS the covr failure).
**Decision:** Workspace mode (triggered when /init-project runs in a non-repo directory containing N git repos; a multi-package single repo is a monorepo, NOT a workspace) produces: (1) a full, self-contained baseline per repo; (2) ONE canonical `<workspace>/.etc_workspace/seam-map.yaml` (schema_version; repos; seams keyed by kind: url-routing | auth-session | data-schema | embed-loader, each with owner_repo/consumer_repos/contract/evidence; workspace confidence) as the single editable source; (3) read-only per-repo mirrors — each repo's baseline `seams:` block is regenerated by `baseline.py sync-seams`, filtered to the seams touching that repo, with a hand-edits-will-be-overwritten header. Seam records with no resolvable sibling require an explicit `boundary-unknown` acknowledgment at ratification and cap confidence at LOW.
**Consequences:** Easier: solo-cloned repos retain full seam context; one place shows the whole system map; per-repo gates work without the workspace. Harder: mirror regeneration is a new consistency obligation — a stale mirror can mislead until the SessionStart freshness hook (check-baseline-fresh.sh) flags drift. Workspace enumeration never crawls upward or follows symlinks out of the workspace.
