# ADR F-2026-06-10-brownfield-architecture-baseline-002: Three-State Baseline Gate — a Scoped Deviation from Advisory-Default

**Date:** 2026-06-11
**Status:** Accepted
**Context:** Layered-review ADR-003 establishes the house gate posture: advisory-default, hard-block only on a recorded escalation flag, because cry-wolf false positives destroy gate trust (F007/F008 lesson). But the covr PBJ failure was a build proceeding against an unverified architecture model, and the operator decided at /spec time (GA-005) that /build must not proceed on an unratified baseline.
**Decision:** A three-state gate in /build (Step 1c sibling), branching on the `baseline.py status` stdout token: `missing` → SOFT path — emit an EXACT verbatim warning naming `/init-project --phase=baseline` (the backfill offer) and PROCEED; `unratified` → HARD STOP naming the ratify command; `malformed` → STOP as infrastructure failure (a corrupt ratification record is never treated as ratified). /spec and /architect warn-with-recorded-override on both non-ratified states. The deviation from advisory-default is scoped and justified: `unratified` is a RECORDED-INTENT state — an operator started ratification and abandoned it — not a heuristic false positive; the ambiguous state (`missing`, where every legacy project lives) keeps the soft path, preserving ADR-003's rationale exactly where it applies. Escape hatch: a repo may be declared `baseline-exempt` with a recorded reason.
**Consequences:** Easier: the covr failure mode becomes structurally impossible; legacy repos feel zero upgrade friction; adoption is a ramp, not a flag day. Harder: an abandoned ratification blocks builds until finished or exempted; future gate authors must not cite this as license to hard-block on heuristics — the deviation applies ONLY to recorded-intent states.
